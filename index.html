ICMP — How Ping Works

ping uses ICMP (Internet Control Message Protocol) — a Layer 3 protocol defined in RFC 792. It sends Echo Request packets (type 8) and waits for Echo Reply packets (type 0). The round-trip time (RTT) is measured in milliseconds.

ICMP Type 8 → Echo Request ICMP Type 0 → Echo Reply TTL = Time to Live (hops) RFC 792 Layer 3 — Network

About This Project
NetCalc — A networking utility tool built as a personal learning project.
Project Overview

NetCalc is a browser-based networking utility I built to strengthen my understanding of IP addressing, subnetting, and network protocols. Rather than just reading theory, I wanted to implement these concepts in working code.

The project covers topics from my networking coursework including IPv4 addressing, CIDR notation, subnet masking, VLSM design, ICMP, and common TCP/UDP ports — all implemented from scratch using pure JavaScript without any libraries.

I use this tool regularly when practicing subnetting problems and during lab exercises.

Concepts Covered
IPv4 Dotted-decimal notation, binary conversion, octet structure
CIDR Classless Inter-Domain Routing, prefix length, supernetting
Subnetting Network/broadcast address, host range, wildcard masks
VLSM Variable Length Subnet Masking, efficient address allocation
ICMP Ping (Echo Request/Reply), TTL, RTT, packet loss
Ports TCP/UDP well-known ports, protocol layers, risk classification
Tech Stack
HTML5 CSS3 Vanilla JavaScript No libraries / frameworks Bitwise IP operations GitHub Pages hosted

All subnet math is done using JavaScript bitwise operators (>>>, &, |, ~) — the same operations a real subnet calculator uses internally.
